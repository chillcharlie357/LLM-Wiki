## http3和之前的版本有哪些区别

| **特性** | **HTTP/1.1** | **HTTP/2** | **HTTP/3** |
| --- | --- | --- | --- |
| **传输层协议** | TCP | TCP | **UDP (通过 QUIC)** |
| **多路复用** | 不支持 (容易阻塞) | 支持 (存在 TCP 级队头阻塞) | **完美支持 (解决队头阻塞)** |
| **连接建立** | 较慢 (TCP + TLS 多次握手) | 较慢 (同前) | **极快 (0-RTT/1-RTT)** |
| **安全性** | 可选 TLS | 事实上强制 TLS 1.2+ | **内置 TLS 1.3 (必须)** |

### **QUIC (Quick UDP Internet Connections)** 核心改进

#### 解决 TCP 队头阻塞 (Head-of-Line Blocking)

- 在 **HTTP/2** 中，虽然多个请求可以共用一个 TCP 连接，但如果其中一个数据包在网络中丢失，TCP 会要求重传，导致整个连接的所有请求都必须等待。
- **HTTP/3 (QUIC)** 改进了这一点：由于它是基于 UDP 的，QUIC 在内部实现了流控制。如果某个流（Stream）丢包，只会影响该流本身，其他流的数据可以继续传输，互不干扰。

#### 极速连接建立 (0-RTT / 1-RTT)

- **传统方式：** 需要先进行 TCP 三次握手，再进行 TLS 握手，通常需要 2-3 个往返（RTT）。
- **HTTP/3：** QUIC 将传输握手和加密握手合并。对于首次连接，只需 1 个 RTT；如果是曾经访问过的服务器，可以实现 **0-RTT** 握手，数据发送几乎是瞬时的。

#### 连接迁移 (Connection Migration)

- **过去：** TCP 连接是基于“四元组”（源IP、源端口、目的IP、目的端口）的。如果你从 Wi-Fi 切换到 4G，IP 变了，TCP 连接就会断开，网页或视频需要重新加载。
- **HTTP/3：** 使用 **Connection ID** 来标识连接。即使你的 IP 地址改变了，只要 ID 没变，连接依然可以保持无缝切换，用户几乎感知不到中断。

#### 内置的安全性 (TLS 1.3)

- QUIC 并不像 TCP 那样把加密当成“可选插件”。它**原生集成**了 TLS 1.3，所有的数据包（除了极少数协商包）都是经过加密和认证的，这比传统的 TCP 更加安全且私密。
