# Go

## 基础数据结构

### Slice（切片）

#### 切片和数组的区别

- 切片在运行时由 `reflect.SliceHeader` 描述。
- `Data`：指向底层数组的指针。
- `Len`：当前切片长度。
- `Cap`：底层数组总容量。

| 特性 | 数组 (Array) | 切片 (Slice) |
| --- | --- | --- |
| 长度 | 固定，定义后不可变 | 动态，可扩容 |
| 类型定义 | 长度是类型的一部分 | 长度不是类型的一部分 |
| 类型本质 | 值类型 | 引用型描述符 |
| 传递方式 | 函数传参拷贝整个数组 | 函数传参传递引用，效率高 |
| 内存分配 | 编译时分配 | 运行时动态分配 |

#### `append` 的扩容机制

- Go 1.18 后，旧容量小于 256 时通常翻倍。
- 旧容量大于 256 时逐步递增并结合内存对齐。
- `append` 可能返回新的切片引用，必须写成 `s = append(s, ...)`。

#### 复制与共享底层数组

- `s1 := s[1:2]` 会共享底层数组。
- 修改元素或未触发扩容的 `append` 可能影响原切片。
- 需要独立副本时显式使用 `copy`。

```go
s2 := make([]int, len(s1))
copy(s2, s1)
```

#### 函数传参

- 切片传递本质上是值传递，传的是切片头副本。
- 未扩容时，元素修改对外可见。
- 触发扩容后，函数内切片指向新内存，对外不可见。
- 长度变化不会反映到外部，除非传递切片指针。

### Map（映射）

- 底层是 `hmap`，通过桶组织数据。
- 每个桶最多存 8 个键值对。
- 扩容是增量完成，不是一次性搬迁。
- Map 遍历无序，起始位置随机。
- 并发读写不安全，需要 `sync.Map` 或互斥锁保护。

### Channel（通道）

- 底层是 `hchan`。
- 有缓冲时通过循环队列存放数据。
- 发送阻塞会把 goroutine 挂到 `sendq`。
- 接收阻塞会挂到 `recvq`。
- Channel 底层操作带锁。

### Interface

- `eface`：空接口。
- `iface`：带方法接口，包含 `itab` 和方法表。
- 类型断言本质上是检查具体类型和接口定义是否匹配。

## 并发模型 GMP 和 Goroutine

- `G`：Goroutine，2KB 栈起步。
- `M`：系统线程。
- `P`：处理器上下文和本地队列。
- Work Stealing：本地队列空时从其他 P 偷一半任务。
- Hand Off：线程阻塞时，P 交给新的 M 继续工作。

## 垃圾回收

- Go GC 采用三色标记 + 混合写屏障。
- STW 主要发生在标记开始和结束阶段。
- 逃逸分析决定对象分配在栈上还是堆上。
- 减少堆分配可以降低 GC 压力。

## 常见工程实践

- `defer`：后进先出。
- `panic` / `recover`：`recover` 必须在 `defer` 中调用。
- `context`：用于超时控制、取消信号和元数据传递。
