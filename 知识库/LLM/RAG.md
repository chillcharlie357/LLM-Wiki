## 检索算法

### 关键词检索算法

#### TF-IDF

- **TF (Term Frequency)**：词频。一个词在文档中出现次数越多，重要性越高。
- **IDF (Inverse Document Frequency)**：逆文档频率。如果一个词在所有文档中都频繁出现（如“的”、“是”），那么它的区分度就很低。

#### BM25

- `score(D, Q) = Σ IDF(q) * f(q, D) * (k1 + 1) / (f(q, D) + k1 * (1 - b + |D| / avgdl))`
- `IDF(q)`：查询词 q 的稀缺程度。
- `f(q, D)`：词 q 在文档 D 中的频率。
- `|D| / avgdl`：当前文档长度与全局平均长度的比值。
- `k1`：控制词频饱和度的速度。
- `b`：控制长度归一化的强度。

#### BM25 对比要点

- 词频饱和度：BM25 用非线性增长避免词频线性膨胀。
- 文档长度归一化：长文档会被适度惩罚，短文档中命中关键词的权重更高。

### 向量检索算法

向量索引算法用于高效查找与给定查询向量最相似的向量，解决海量高维空间里的最近邻搜索性能问题。

#### HNSW

HNSW (Hierarchical Navigable Small World) 可以理解成一个多层高速公路网络：

```text
Layer 2: A <---------------- 50km ----------------> B
         |                                          |
Layer 1: A <----- 20km -----> C <----- 20km -----> B
         |                    |                     |
Layer 0: A <-> 5km <-> D <-> 5km <-> C <-> 5km <-> E <-> 5km <-> B
```

- 层次化构建：高层节点更少，形成金字塔结构。
- 贪心搜索：从顶层开始逐层逼近最近邻。
- 由粗到精：上层快速定位区域，下层精确检索。

## Hybrid / 混合检索

- 关键词检索擅长精确匹配，适合专有名词。
- 向量检索擅长语义理解，适合模糊表达。
- 混合检索会并行执行两者，再做去重、融合和重排序。

### 合并策略

- 线性组合：给关键词检索和向量检索手动加权。
- RRF（Reciprocal Rank Fusion）：文档在多个结果列表里排名越靠前，最终分数越高。

## 为什么要有向量数据库？和传统数据库的区别

- 传统关系型数据库擅长精确查询和范围查询。
- 向量数据库使用 HNSW、IVF 等索引算法高效处理相似度查询。

## Rerank

### 为什么要 rerank

1. 首次检索是粗排，追求广度和效率。
2. Rerank 是精排，用 rerank 模型对 top-k 结果重新打分排序。

### 为什么首次检索不够

1. 向量语义偏移，可能相似但不相关。
2. top-k 是经验值，关键片段可能落在召回边界外。
3. 上下文有限，不精排会浪费上下文窗口。
4. 高质量片段可能因为初排靠后被忽略。

### Rerank 模型

- 基于 cross-encoder，把 query 和 document 拼接起来判断真实相关性。
- 代表模型：BGE-Reranker、Cohere Rerank、Jina Reranker。

## 如何处理 PDF、Markdown、Word 等不同格式文档

1. 自己写代码逐种处理，灵活但成本高。
2. 使用成熟的 Document Loader 把各种格式转成统一文本后再分块和向量化。

### Data-Juicer

- 阿里巴巴开源的数据处理工具，可通过 YAML 灵活配置算子。
- 文档标准化、清洗、质量过滤、去重都可以在流水线上完成。

## Query 重写

### 为什么要重写

- 用户原始 query 可能模糊、口语化、信息缺失或意图混杂。

### 常见策略

1. HyDE：先生成假设答案，再把它转成向量去检索。
2. Multi-Query：把一个问题从多个角度改写并并行检索。
3. Step-back Prompting：先退一步生成更抽象的问题。
4. Query Decomposition：把复杂问题拆成多个子问题。
5. Reverse HyDE：从文档块生成假设问题再做匹配。
