# 消息队列

## Kafka 为什么高性能

Kafka 高性能 = **顺序写** + **Page Cache** + **零拷贝** + **批量压缩** + **分区并行**

### 1. 顺序读写

Kafka 采用 append-only 日志写入。

- 顺序 I/O 比随机 I/O 快得多。
- 避免磁盘频繁寻道和旋转延迟。

### 2. 零拷贝

Kafka 使用 `sendfile`，把数据直接从页缓存传到 NIC 缓冲区，减少用户态参与。

传统 I/O 通常会经历 4 次拷贝和 4 次上下文切换，零拷贝显著降低 CPU 和内存复制开销。

### 3. 页缓存

- Kafka 直接利用操作系统 Page Cache。
- 减少 JVM 堆内缓存，降低 GC 压力。
- 热数据可在 Broker 重启后继续被系统缓存命中。

### 4. 批量处理

- Producer 批量发送。
- Consumer 批量拉取。
- 降低网络请求数和协议开销。

### 5. 分区与并发

- Topic 被划分为多个 Partition。
- 同一 Consumer Group 内可对多个分区并行消费。

## Kafka vs RabbitMQ vs Redis Stream

| 特性 | Kafka | RabbitMQ | Redis Stream |
| --- | --- | --- | --- |
| 设计核心 | 分布式日志存储 | 消息代理 | 内存数据结构 |
| 存储模型 | 磁盘顺序日志 | 队列模型 | 内存存储，可持久化 |
| 数据消费 | Pull | Push | Pull |
| 吞吐量 | 极高 | 中等 | 高 |
| 延迟 | 毫秒级 | 微秒级 | 微秒级 |
| 数据顺序 | 分区内有序 | 队列内有序 | ID 递增有序 |
| 场景 | 日志采集、流计算 | 金融交易、复杂路由 | 轻量异步任务、实时通知 |
