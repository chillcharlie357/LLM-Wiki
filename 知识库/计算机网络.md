# 计算机网络

## HTTP/3 和之前版本的区别

| 特性 | HTTP/1.1 | HTTP/2 | HTTP/3 |
| --- | --- | --- | --- |
| 传输层协议 | TCP | TCP | UDP (通过 QUIC) |
| 多路复用 | 不支持 | 支持，但有 TCP 级队头阻塞 | 完美支持 |
| 连接建立 | 较慢 | 较慢 | 极快 (0-RTT / 1-RTT) |
| 安全性 | 可选 TLS | 事实上强制 TLS 1.2+ | 内置 TLS 1.3 |

## QUIC 核心改进

### 解决 TCP 队头阻塞

- HTTP/2 虽支持多路复用，但丢包会阻塞整条 TCP 连接。
- HTTP/3 基于 UDP，在 QUIC 内部实现流控制，单个流丢包不会拖住其他流。

### 极速连接建立

- 传统模式需要 TCP 三次握手加 TLS 握手。
- QUIC 把传输握手和加密握手合并，首次连接只需 1 RTT，复用连接可做到 0 RTT。

### 连接迁移

- TCP 依赖四元组，网络切换时连接会断。
- QUIC 使用 Connection ID，IP 变化后连接仍可保持。

### 内置安全性

- QUIC 原生集成 TLS 1.3。
- 大部分数据包默认加密与认证，安全性更强。
