# G1

- 内存分为多个Region
    - Region逻辑上划分为**Eden，Survivor和老年代**。每个分区都可能是eden区，Survivor区也可能是old区，但在一个时刻只能是一种分区。
    - **Humongous**区，存放大对象
    - Region逻辑上连续，物理上不连续
- 可以建立预测停顿时间模型

- GC类型
    - Minor GC：回收eden和survive
    - Mixed GC：回收完整内存
- GC过程
    - 标记
    - 复制
    - 清理

# ZGC

- 内存划分为一系列page
    - JDK21版本ZGC加入分代
- 保证最大停顿时间在几毫秒之内，无论堆大小/对象多少
- 可以处理 8MB-16TB 的堆

![image.png](%E5%9E%83%E5%9C%BE%E5%9B%9E%E6%94%B6%E5%99%A8/image%202.png)

- GC过程
    - 标记：从gc root开始标记存活对象
    - 转移：选择部分活跃对象转移到新的内存空间上
    - 重定位：因为对象地址变了，所以之前指向老对象的指针都要换到新对象地址上。
