# 基础概念

## OOP

### **虚方法表**（**vtable**）

在 **JVM 加载类的时候**，会为每个类创建一个 vtable。这张表本质上是一个数组，里面存储了该类所有**虚方法**（非 `static`、非 `final`、非 `private` 的方法）的实际入口地址。

- **继承与重写**：如果子类重写了父类的方法，那么子类 vtable 中对应位置的地址就会指向子类自己的实现。
- **沿用父类**：如果子类没有重写，那么该位置的地址依然指向父类的实现。

### **非虚方法**

JVM 对它们的处理方式与普通方法不同：

- **静态方法（`static`）**：属于类而不属于对象，调用时直接指向类的内存地址。
- **私有方法（`private`）**：子类无法继承，更谈不上重写，所以调用目标是唯一的。
- **最终方法（`final`）**：明确禁止子类重写。既然只有这一个版本，JVM 在编译期就能确定该调哪段代码。

### **抽象类（Abstract Class）** 与 **接口（Interface）** 的区别

| 特性 | 抽象类 (abstract class) | 接口 (interface) |
| --- | --- | --- |
| 成员变量 | 可以有各种类型的变量（静态、实例、常量） | 默认是 public static final (常量) 💎 |
| 方法实现 | 可以有普通方法，也可以有抽象方法 | Java 8 后支持 default 和 static 方法 🛠️ |
| 构造函数 | 有（供子类初始化使用） | 没有 🚫 |
| 继承/实现 | 单继承（只能有一个父类） | 多实现（可以实现多个接口） 🔗 |
| 设计语义 | 强调 "is-a"（是一类事物的抽象） | 强调 "has-a / can-do"（具备某种能力） |

虽然 Java 8 引入了 `default` 方法，让接口看起来像是在“做事”，但由于接口不能定义**非静态的成员变量（实例变量）**，它依然无法拥有自己的“状态”。

- **抽象类：** 它可以拥有私有的成员变量（如 `private String name;`），并通过构造函数初始化。它的方法可以根据这些变量的状态改变行为。
- **接口：** 即使有了 `default` 方法，它也只能利用传入的参数或者调用其他抽象方法来工作。它没有属于自己的“记忆”（实例变量）。

### 为什么抽象类不能实例化，但是有构造方法？

抽象类构造函数存在的真正意义在于：**初始化从父类继承下来的成员变量**。

### 构造函数的调用链

```java
class Animal {
    public void shout() { System.out.println("动物叫"); }
}

class Cat extends Animal {
    @Override
    public void shout() { System.out.println("喵喵喵"); }
    
    public void catchMouse() { System.out.println("抓老鼠"); }
}

// 测试代码
Animal myCat = new Cat();
myCat.shout();      // 语句 A
myCat.catchMouse(); // 语句 B
```

当我们执行 `new Cat()` 时，尽管我们不能直接 `new` 一个抽象类，但内存中确实存在一个完整的 `Cat` 对象，它包含了 `Animal` 部分和 `Cat` 特有的部分。

1. **父类初始化**：子类的构造函数默认会调用 `super()`。这样可以确保父类中定义的属性（比如 `Animal` 的 `age` 或 `name`）被正确赋值。
2. **子类初始化**：在父类初始化完成后，再执行子类特有的初始化逻辑。

## String、StringBuilder 与 StringBuffer

- **String**：**不可变**（底层是 `final` 修饰的数组）。每次修改都会创建新对象，浪费内存。
- **StringBuilder**：**可变**，效率最高。但它是**线程不安全**的，适用于单线程。
- **StringBuffer**：**可变**，方法加了 `synchronized`。它是**线程安全**的，但性能略低。

> **面试：** String 为什么设计成不可变？（安全性、哈希码缓存、字符串常量池优化）。

## 泛型 (Generics) 与 类型擦除

**泛型（Generics）** 允许你在编写代码（如类、接口、方法）时，不预先指定具体的常量或类型，而是将**类型**作为一个**参数**。

简单来说，泛型就是“**类型的参数化**”。

Java 的泛型是“伪泛型”，这是为了兼容老版本 Java。

- **类型擦除**：泛型信息只存在于**编译阶段**。一旦程序运行（进入 JVM），所有的泛型信息都会被擦除，转为 `Object` 或限定类型。
- **副作用**：你不能在运行时通过 `instanceof T` 来判断泛型类型，也不能 `new T()`。

> **面试题：** 既然运行期会被擦除，那为什么 `List<String>` 不能存 `Integer`？（因为编译器在编译阶段会插入**类型检查**和**强制转换**的代码）。

## 异常处理体系 (Exception)

Throwable 的家族树：

- **Error**：JVM 无法处理的严重问题（如 `OutOfMemoryError`），程序通常直接挂掉。
- **Exception**：程序可以处理的异常。
    - **Checked Exception (编译时异常)**：必须 `try-catch` 或 `throws`（如 `IOException`）。
    - **Runtime Exception (运行时异常)**：编译器不强制要求处理（如 `NullPointerException`）。

### Object 类与基本约定

所有的类都默认继承自 `Object` 类。

- **`==` 与 `equals()`**：
    - **`==`**：对于基本类型，比较的是**值**；对于引用类型，比较的是**内存地址**。
    - **`equals()`**：默认行为和 `==` 一样（比较地址）。但很多类（如 `String`、`Integer`）会重写它来比较**对象的内容**。
- **`hashCode()`**：返回对象的哈希码。它和 `equals` 必须遵循一个原则：**如果两个对象 `equals` 相等，那么它们的 `hashCode` 也必须相等**。

# 集合Collections

主要分为两类：

1. **Collection 接口** ：用于存放单一元素的容器。它下面有三个著名的子接口：
    - **List**：像排队一样，元素是有序的，且允许重复（如 ArrayList）。
    - **Set**：像一个特立独行的俱乐部，元素无序，且不允许重复（如 HashSet）。
    - **Queue**：像超市收银台，主要用于遵循“先进先出”（FIFO）原则的操作。
2. **Map 接口** ：用于存放“键值对”（Key-Value）的容器。它通过一个唯一的“键”来映射到一个特定的“值”，就像字典里通过单词查解释一样（如 HashMap）。
